# やよいの青色申告オンライン CSV 生成スキル

領収書・レシート等の取引データを、やよいの青色申告オンライン／弥生会計オンラインの「スマート取引取込」でインポートできる CSV に変換する。

## 必ず守る制約

- **文字コード: Shift-JIS**（UTF-8 では取込不可）
- **改行コード: CR+LF**
- **1 ファイル < 200 KB**
- **入金 / 出金は 9 桁以内**、**摘要は 80 文字以内**
- 西暦は **4 桁** で書く（`22/04/01` は令和 22 年扱いになる）
- 1 取引 = 1 行（決済単位でまとめ、商品名は摘要にカンマで列挙）

## 推奨ワークフロー

文字コードや CRLF を取り違えると弥生側で「読み込めない行があります」と弾かれる。手書きで CSV を組まず、必ず同梱のスクリプトで出力すること。

1. ソース（PDF・画像・明細）から取引を抽出する。1 取引につき:
   - 日付 / 入金額 / 出金額 / 摘要 / 軽減税率対象か（食品・飲料は対象）
2. 取引を JSON 配列にまとめる（[scripts/build_csv.py](scripts/build_csv.py) のヘッダーに入力スキーマあり）
3. `scripts/build_csv.py` に渡して Shift-JIS / CRLF の CSV を出力する:

   ```bash
   python3 scripts/build_csv.py < transactions.json > out.csv
   ```

4. ファイルサイズが 200 KB を超える場合は期間で分割する

## サンプル

ヘッダーあり・2 列方式（推奨）:

```csv
日付,入金,出金,摘要,軽減税率,部門,請求書区分
2024/06/15,,3280,さくら電器 ワイヤレスマウス,,,
2024/08/22,,1500,丸山書店 プログラミング入門書,,,
```

実物のサンプル（Shift-JIS / CRLF）は [examples/](examples/) を参照。
